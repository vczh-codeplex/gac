Introduction

Create a GacUI program is pretty easy. GacUI is released by source code, what you need to do is adding these 5 source files into your project, and write WinMain and GuiMain, then it is done! If you are new to GacUI, we recommended you to follow the instructions below to build your first GacUI program:

  • Step 1. Get the GacUI source code.
  • Step 2. Create a Visual C++ Project.
  • Step 3. Write your first GacUI program.
  • Step 4. Build and run!

Get the GacUI source code.

GacUI provides two way to get the source code. The first one is to download the release. The second one is to download the latest source code. If you want to rely on a well tested and stablized GacUI version, use the release. If you want to try some unreleased feature, try the latest source code.

The GacUI project hosts in http://gac.codeplex.com/. If you don't know how to download them, please go to the DOWNLOAD page.

Create a Visual C++ Project.

We recommend Visual Studio 2010 as the IDE to create a GacUI project. GacUI implementation doesn't use any C++11 feature, so Visual Studio 2008 is ok, but we have not spend enough time testing it yet.

If you don't know how to create a Visual C++ project, please read this MSDN article.

After a Visual C++ project is created, add these 5 source files to your project. If you still have trouble getting these files, please read the DOWNLOAD page again.

  • GacUIIncludes.h
  • GacUI.h
  • GacUI.cpp
  • Vlpp.h
  • Vlpp.cpp

Precompiled headers are recommended to use for a GacUI project. Adding GacUIIncludes.h or GacUI.h to your precompiled headers will speed up the compiling, especially for big projects. Remember disable precompiled headers for GacUI.cpp and Vlpp.cpp, or you have to add #include "stdafx.h" to these 2 cpp files yourself.

Write your first GacUI program.

The following code can be found in Libraries\GacUI\GacUIDemo\HelloWorld\Main.cpp if you DOWNLOAD the latest source code.

Even a shortest GacUI program should contains 2 functions: WinMain and GuiMain. If you don't know how to write a WinMain function for a Windows program, please read this MSDN article.

WinMain

The only function you need to call in WinMain is SetupWindowsDirect2DRenderer or SetupWindowsGDIRenderer. This is the only chance to choose which renderer to use after a program is executed. If you want to use Direct2D which is a good solution to speed up your program using GPU, call SetupWindowsDirect2DRenderer. If you want to use GDI, which is only suggested to use in an environment that Direct2D is not available, call SetupWindowsGDIRenderer. If you want to make your program compatible with Windows XP operating system, you can test if the system supports Direct2D in WinMain, and then decide which function to call.

GuiMain

The other function is GuiMain. A GuiMain function is required to be a GacUI program. When the GacUI finishes all the initializations, GuiMain function will be called, and you can use GetApplication()->Run to run a main window. GacUI supports multiple windows at the same time, but you should have a main window. GacUI will start to do finalization when the main window is closed.

The following code shows how to write a hello world using GacUI.

Hello, world!

#include "..\..\Public\Source\GacUIIncludes.h"
#include <Windows.h>

int CALLBACK WinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance, LPSTR lpCmdLine, int CmdShow)
{
    return SetupWindowsDirect2DRenderer();
}

void GuiMain()
{
    GuiWindow* window=g::NewWindow();
    window->SetText(L"Hello, world!");
    window->SetClientSize(Size(640, 480));
    window->MoveToScreenCenter();
    GuiControl* label=g::NewLabel();
    label->SetText(L"Welcome to GacUI Library!");
    {
        FontProperties font;
        font.fontFamily=L"Segoe UI";
        font.size=40;
        font.antialias=true;
        label->SetFont(font);
    }
    window->AddChild(label);
    GetApplication()->Run(window);
    delete window;
}
